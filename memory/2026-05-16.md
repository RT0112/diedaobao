

### 23:20 - APK安装成功突破！通过am start绕过MIUI限制

**根因**：`adb install` / `pm install` 被MIUI静默拦截（INSTALL_FAILED_USER_RESTRICTED），但 `am start` 触发系统安装器可以正常弹出安装界面。

**正确的安装流程（三步弹窗）**：
1. `adb push` APK到 `/sdcard/Download/`（不能用 `/data/local/tmp/`，am start读不到）
2. `adb shell am start -a android.intent.action.VIEW -d file:///sdcard/Download/diedaobao-debug.apk -t application/vnd.android.package-archive`
3. MIUI安装器弹出三步确认：
   - ①ICP备案弹窗：点"继续安装"（左侧按钮！右侧是"退出"）
   - ②安全提醒页：勾选"已了解此应用未经安全检测"CheckBox → 点"继续安装"
   - ③指纹验证：需要用户在手机上按指纹

**关键经验**：
- MIUI "USB安装"开关一直是关的也能装，关键是走 `am start` 系统安装器，不要走 `adb install`
- `file:///sdcard/Download/` 路径OK，`file:///data/local/tmp/` 会被packageinstaller拒绝(packageInfo is null)
- ICP备案弹窗的"继续安装"在**左侧**（容易误点右侧"退出"）
- 安全提醒页必须先勾选CheckBox才能点继续安装
- 最后一步指纹验证必须物理操作，无法通过ADB绕过
- 之前全能开发2的 `adb install -r` 能成功，可能是因为当时有旧版app在（升级模式不触发指纹验证？），或者当时MIUI版本不同

**当前状态**：安装界面停在指纹验证步骤，待用户在手机上按指纹完成安装

### 23:35~23:44 - K70 Termux SSH配置尝试

K70已上线(192.168.4.105)，开始配置SSH环境以便后续部署后端。

**遇到的问题**：
- 通过SSH连接K70 Termux后运行 `termux-setup-storage`，提示符要求输入 `y/n` 确认
- **`input text y` + `input keyevent 66`(回车) 都被Termux当成shell命令执行**，显示 `y: not found`
- **`input keyevent KEYCODE_CTRL_C` (Ctrl+C) 也无效**，提示符仍然卡在等待输入状态
- 根因：Termux的shell会把 `input text/keyevent` 的内容当作命令来解析执行，而不是作为stdin输入给 `termux-setup-storage`

**教训/待解决**：
- Termux交互式命令的自动化输入是个难题 — ADB input机制在Termux内不work
- 可能需要换方案：比如用 `expect`、或者直接跳过 `termux-setup-storage` 手动创建目录、或者用其他方式传递公钥
- 存储权限实际上可能已经存在（之前装过），可以尝试直接操作 `/sdcard/` 目录
- 会话在尝试多种input方式后被自动中止（步骤过多）

### 23:52~23:54 - 方案2(Termux:API RunCommand)深入分析

用户问方案2是否意味着可以直接输入命令并读取结果。

**关键发现**：
- RunCommand通过 `am start --es com.termux.RUN_COMMAND '命令'` 执行，能解决**shell用户≠Termux用户**权限问题
- 但 `am start` 不返回stdout，无法直接读取命令输出
- 变通：命令重定向到 `/sdcard/output.txt`，然后 `adb pull` 读取

**核心策略**：RunCommand不是最终方案，而是**打开SSH的钥匙**
1. 装Termux:API APK（am start安装器，已验证可行）
2. RunCommand执行：`mkdir ~/.ssh && cp /sdcard/k70_key.pub ~/.ssh/authorized_keys`
3. SSH通了之后一切走SSH — 完整交互式shell，输入输出全通

**待执行**：下载Termux:API APK → push到K70 → 安装 → RunCommand配置SSH → SSH直连部署server.js
