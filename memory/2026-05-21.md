## 07:21 隧道恢复 - ngrok+SSH重启，双端外网连接成功

### 问题
- 早上用户报告连不上（ngrok和SSH隧道 overnight 都断了）
- SSH隧道进程还在但连接已死，ngrok进程被session结束时kill了

### 解决
1. 杀掉旧的SSH隧道进程，重建：`ssh -f -N -L 13000:localhost:3000 ...`
2. 用 `nohup + disown` 启动ngrok，防止session结束时被杀：
   ```bash
   nohup /Users/zhou/ngrok http 13000 &>/tmp/ngrok.log & disown
   ```
3. ngrok URL仍为 `https://oscular-august-equanimously.ngrok-free.dev`
4. 重启K70上双端App连接ngrok外网

### 验证
- `/health` → `{"status":"ok"}`
- elders: 1, guardians: 1 — 双端都通过外网连上了

### 经验
- ngrok必须用nohup+disown启动，否则session结束时被kill
- SSH隧道overnight可能断开，需要重建
- Mac睡眠/关机也会断隧道
