# 连续查看位置测试 — 2026-05-22

## 测试目标
用户要求：连续点击"查看位置"几次，看是否有问题。

## 测试环境
- K70设备：a0c2910e
- 老人端：v0.45.6 (versionCode=148)
- 子女端：v0.8.7 (versionCode=31)
- ADB路径：/Users/zhou/Library/Android/sdk/platform-tools/adb

## 测试方法
1. 清logcat → 点"查看位置" → 等5秒 → 抓日志 → 返回主页
2. 重复3次

## 测试结果

### 子女端（MapActivity）
- ✅ 3次都能正常打开地图页面
- ✅ 地图正常渲染，有红色位置标记
- ❌ 但位置数据是旧的（pullLocationStatus永远pending）

### 老人端（崩溃！）
- ❌ 第1次点击 → **CRASH** — 老人端进程崩溃
- ❌ 第2次点击 → **CRASH** — 系统标记"Too noisy! skip duplicate"
- ❌ 第3次点击 → **CRASH** — 同上

### 关键错误（每次都重现）
```
java.lang.NoSuchMethodError: No virtual method put(Ljava/lang/String;F)Lorg/json/JSONObject;
at com.falldetector.diedaobao.cloud.CloudBaseClient.uploadLocationNow(CloudBaseClient.kt:337)
```

## 根因分析（给CC的完整信息）
- 文件：`fall-detection-app/.../cloud/CloudBaseClient.kt` 第337行
- 错误：用了 `JSONObject.put(String, Float)` —— Android JSONObject 没有这个签名
- 影响：老人端每次收到 pull 位置请求就崩溃，无法上传位置
- 结果：子女端看到 `pullLocationStatus: "pending"` 永远不变

## 修复要求
1. 把 `CloudBaseClient.kt:337` 的 `JSONObject.put(String, Float)` 改成 `put(String, Double)`
2. **必须编译验证通过**（这是CC违反的铁律）
3. 重新打包老人端APK

## 教训
- CC上次修位置精度问题时引入了这个bug
- CC修完后没有强制要求编译验证
- JSONObject.put(Float)在Android上编译通过但runtime crash
- skill已更新：qclaw-cc-integration增加"编译验证是强制的"原则 + JSONObject.put(Float)陷阱
