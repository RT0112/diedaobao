## 22:40 Session — v0.19.1 整体决策优化

### 用户反馈与决策
1. 用户反馈「举着手机看到放进口袋依旧误报」→ 需要整体优化决策链
2. 用户要求「整体优化决策」→ 从 ML 主导改为物理特征主导

### 根因分析
**场景**：举着手机看 → 逐渐降低到口袋 → 松手落入口袋

**误报路径**：
- 自由落体（松手后）时间：140-200ms
- 冲击峰值：1.5-2.5g（硬口袋底可能 ≥2.0g）
- v0.18.2 代码：`impactTooLow < 2.0f` → 硬口袋底可能 ≥2.0g → `isPocketLikely = false` → 进入 ML → 误报

**根因**：口袋排除阈值太松（`impactTooLow < 2.0f`），ML 权重过高（物理特征 0%，ML 100%）

### 优化方案（v0.19.1）

#### 1. 收紧口袋排除阈值
| 参数 | 旧值 | 新值 | 理由 |
|------|------|------|------|
| `impactMinG` | 2.0f | **2.5f** | 口袋冲击 < 2.5g，真实跌倒 > 2.5g（硬地面）|
| `ffTimeMs` | 180ms | **200ms** | 口袋自由落体 < 200ms，真实跌倒 > 250ms |

#### 2. 加强物理规则权重
**旧逻辑**：`fallScore = ffScore * 0.30 + impScore * 0.35 + velScore * 0.35`（ML 0%，物理 100%）
**新逻辑**：
```kotlin
physScore = ffScore * 0.35 + impScore * 0.35 + velScore * 0.30
fallScore = if (mlRaw >= mlLow) {
    physScore * 0.7 + mlRaw * 0.3   // ML 只占 30%，物理特征主导
} else {
    physScore * 0.9                // ML 不可用，完全依赖物理特征
}
```

**效果**：ML 从「决策者」降级为「参考者」，物理特征（自由落体时间、冲击峰值、速度）占 70% 权重。

#### 3. 主链也检查口袋特征
**旧逻辑**：只有自由落体分支检查口袋特征，主链（非自由落体）直接进 ML。
**新逻辑**：在 `runMlDecisionTree` 开头加检查：
```kotlin
if (freefallTimeMs < 100 && peakValue < cfg.impactMinG) {
    if (mlRaw < 0.95f) {  // 没有极高 ML 置信度 → 排除
        推送诊断（主链排除）
        return
    }
}
```

**效果**：即使自由落体分支漏检，主链也有第二道防线。

**问题**：手机从向上（屏幕朝外）到向下（屏幕朝内），角度变化 60-180°，ML 概率偏高。

**解决方案**：加入角度变化检测，对角度变化大的事件进行惩罚。

**逻辑**：
- 记录事件开始时的重力方向角度
- 计算事件期间的角度变化 `angleChange`
- 如果 `angleChange > 60°` 且 `ffTimeMs < 400ms` → 物理分打折（最多5折）

**代码关键点**：
```kotlin
// 计算角度变化
val startAngle = atan2(gravity[2], sqrt(gravity[0]*gravity[0] + gravity[1]*gravity[1]))
val endAngle = atan2(eventEndGravity[2], sqrt(eventEndGravity[0]*eventEndGravity[0] + eventEndGravity[1]*eventEndGravity[1]))
val angleChange = abs(endAngle - startAngle) * 180 / PI

// 角度变化惩罚
if (angleChange > 60 && ffTimeMs < 400) {
    physScore *= 0.5f  // 物理分打折
}
```

#### 2. FF 质量检测（区分真假自由落体）
**问题**：手降低手机（渐进减速）vs 真自由落体（松手掉落），FF 时间可能相同，但加速度模式不同。

**解决方案**：计算 FF 期间的平均加速度大小 `avgFfMag`，判断是否为真自由落体。

**逻辑**：
- 真自由落体：`avgFfMag ≈ 0g`（失重）
- 手降低手机：`avgFfMag ≈ 0.3-0.5g`（渐进减速）

**代码关键点**：
```kotlin
// 计算 FF 质量
var ffAccMagSum = 0f
var ffAccMagCount = 0
for (i in ffStart..ffEnd) {
    val mag = sqrt(accelerometerData[i][0]*accelerometerData[i][0] + ...)
    ffAccMagSum += mag
    ffAccMagCount++
}
val avgFfMag = if (ffAccMagCount > 0) ffAccMagSum / ffAccMagCount else 9f
val ffQuality = max(0f, 1.0f - avgFfMag / 2.0f)  // 0-1，1=完美自由落体

// FF 质量检查
if (ffTimeMs in 150..300 && ffQuality < 0.6f) {
    // 可能是手降低手机，不是真自由落体
    isPocketLikely = true
}
```
