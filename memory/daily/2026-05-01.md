## 2026-05-01 上午（08:38）

### CloudBase 围栏云函数开发

**目标**：为子女端添加地理围栏功能，老人进出围栏时触发通知

**已完成**：
1. 创建 `geofence` 云函数，支持 CRUD 操作：
   - `create`: 创建围栏（含名称、中心点、半径、通知类型）
   - `list`: 查询某老人的所有围栏
   - `update`: 更新围栏
   - `delete`: 删除围栏
   - `check`: 检查点是否在围栏内（供 location-sync 调用）

2. HTTP 路由已配置：`/geofence`

3. 数据库集合：`geofences`

**部署问题**：
- 函数调用返回错误：`{"code":443, "message":"0 code exit unexpected"}`
- 状态码 443 通常是 TLS/SSL 或网络错误
- **诊断结论**：`scf_bootstrap` 文件可能不适用于 CloudBase Functions
  - CloudBase Functions 与 SCF 原生函数的启动机制不同
  - `get-status` 等 HTTP 函数没有 `scf_bootstrap`，直接用 `index.main` 入口

**待验证修复**：
- 删除 `geofence/scf_bootstrap` 文件
- 重新部署并测试

**关键发现**：
- CloudBase HTTP 函数入口应为 `index.main`，不需要 `scf_bootstrap`
- `tcb fn invoke` 需加 `-e diedaobao-cdn-d4g496tvv296f0ac2` 指定环境

**文件位置**：
- 云函数代码：`projects/fall-detection-app/cloudbase/functions/geofence/index.js`
- 依赖：`projects/fall-detection-app/cloudbase/functions/geofence/package.json`
- 配置：`projects/fall-detection-app/cloudbase/functions/geofence/cloudbaserc.json`

## 2026-05-01 上午（11:06）— 电子围栏Tab闪退修复 ✅

### 闪退现象
用户绑定账号后点击"电子围栏" Tab 闪退

### 根因分析
- **问题代码**：`GeofenceFragment.loadFences()` 和 `deleteFence()` 使用 `lifecycleScope`
- **Fragment 生命周期陷阱**：
  - `lifecycleScope` 绑定 Fragment 生命周期
  - Fragment 的 View 可能被销毁重建（返回栈、旋转屏幕等）
  - 此时 `_binding` 为 null，但协程仍在执行
  - 访问 `b.xxx` 触发 NPE

### 修复方案
```kotlin
// 错误写法
lifecycleScope.launch { ... }

// 正确写法（Fragment 中）
viewLifecycleOwner.lifecycleScope.launch {
    _binding?.let { b ->
        // 所有 binding 操作
    }
}
```

### 修改文件
- `GeofenceFragment.kt`
  - `loadFences()`: `lifecycleScope` → `viewLifecycleOwner.lifecycleScope`
  - `deleteFence()`: `lifecycleScope` → `viewLifecycleOwner.lifecycleScope`
  - 所有 binding 操作加 `_binding?.let` 防御

### 版本
- 亲情守护 v0.6.3 (commit 97d1119)
- APK: 6.7M

### 技术教训
- **Fragment 协程铁律**：永远用 `viewLifecycleOwner.lifecycleScope`，不用 `lifecycleScope`

### 闪退现象
亲情守护 v0.6.3 绑定后点击电子围栏 Tab 闪退（上次修复引入新问题）

### 根因分析
**问题代码**：`private val b get() = _binding ?: throw IllegalStateException("View destroyed")`

- `GeofenceFragment` 有4个地方（HomeFragment等也有）用了这个 throw getter
- 协程执行时 Fragment View 已销毁，`_binding` 为 null
- 访问 `b.xxx` 时 throw getter 抛异常 → 闪退
- 这是 **上次闪退修复引入的 Bug**：移除协程内 `_binding?.let` 后，换成了 throw getter

### 修复方案
**方案1（推荐）**：移除 throw getter，改为优雅降级
```kotlin
// 错误
private val b get() = _binding ?: throw IllegalStateException()

// 正确
private val b get() = _binding ?: return
```

**方案2**：协程内用 `?.let` 安全访问
```kotlin
_binding?.let { b ->
    // binding 操作
}
```

### 闪退问题
用户绑定账号后点击"电子围栏" Tab → 闪退

### 根因
`CloudBaseClient.addGeofence()` 返回 `Boolean`：
- 网络错误（非200）→ `false`
- 云端返回错误（"缺少 elderId"）→ `false`
- **静默失败**，用户只看到"添加围栏失败，请重试"

子女端从未调用 `user-register`（自动注册），所以 `userId = null`，`elderId = null`。
→ 云函数 `createFence` 校验失败，返回 `"缺少 elderId"`
→ 客户端只收到 `false`，用户完全不知道原因

### 修复方案
`addGeofence()` 返回值从 `Boolean` → `String`：
- `""`（空字符串）= 成功
- 具体错误消息 = 失败原因

云函数实际返回的 `message` 字段被充分利用。

| 错误类型 | 用户看到的提示 |
|----------|--------------|
| elderId为空 | "请先在首页绑定老人设备" |
| 网络错误 | "网络错误(500)" |
| 云端拒绝 | 原始错误消息 |

- 根本跑不了，安装后可能闪退或功能异常

### 修复方案

**云函数修复**：

```javascript
// bind-family/index.js
// 同时接受 guardianId 和 familyId
const familyId = data.familyId || data.guardianId;
```

```kotlin
splits {
    abi {
        isEnable = true
        reset()
        include("armeabi-v7a", "arm64-v8a", "x86", "x86_64")
        isUniversalApk = true
    }
}
```
