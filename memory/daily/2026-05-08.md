## MP授权令牌复用方案 — 用户提议，已分析

### 用户提议
保存授权令牌(resultCode+Intent)，下次直接用 `getMediaProjection()` 重建MP实例。

### 结论：❌ 不可行
Android安全机制设计：授权令牌是一次性的，`getMediaProjection(resultCode, data)` 调用后令牌即销毁，无法复用。

### 替代可行方案：不释放VirtualDisplay，只暂停帧上传
- 协助结束时不调 `mediaProjection.stop()`，只停止帧采集上传
- VirtualDisplay 保持活着，下次协助直接恢复采集

## CSDN搜索结果：AccessibilityService自动点击MP弹窗 — 已验证可行

### 来源
CSDN guodashen007文章，在小米5s Plus/Redmi 5A/华为Mate 10上验证通过。

### 三种方法
1. **文本匹配**：`findAccessibilityNodeInfosByText("立即开始")` + `performAction(ACTION_CLICK)`，华为为"允许"
2. **resource-id匹配**：`findAccessibilityNodeInfosByViewId("android:id/button1")` + `performAction`，确认按钮ID固定
3. **对称坐标计算**：当只能找到"取消"(`android:id/button3`)时，通过其bounds镜像计算"允许"位置，用dispatchGesture手势点击

### 关键代码要点
- 监听事件需过滤包名 `com.android.systemui`
- 判断类名包含 `MediaProjectionPermissionActivity`
- 华为mate10可能出现 `event.getSource()` 为null，需改变页面状态重新获取节点
- accessibility-service xml需配置 `canPerformGestures="true"`、`canRetrieveWindowContent="true"`
- dispatchGesture需在子线程中用Handler+Looper调用

## v7方案提议：智能多步弹窗处理（待用户确认）

### 背景
v5现有PermissionRecordManager逻辑基本对路，但两个缺陷导致K70失败：
1. POSITIVE_KEYWORDS缺少"下一步"/"共享屏幕"→找不到正面按钮
2. 不支持Spinner→K70需先切"共享整个屏幕"再点确认

### v7核心思路（无需录制回放）
1. 检测到弹窗 → 扫描节点树
2. 发现Spinner(`screen_share_mode_spinner`) → 点击展开 → 找"共享整个屏幕"选项点击
3. 发现button1(`android:id/button1`) → 点击确认
4. 循环检测 → 弹窗还在就继续处理

### 为什么不需要录制回放
- `screen_share_mode_spinner` 是AOSP标准resource-id，不是MIUI特有
- `android:id/button1` 所有Android都有
- 直接硬编码AOSP标准特征即可，不需要首次学习

### 用户尚未确认选v7还是录制回放方案

- 不同ROM弹窗结构不同，硬编码resource-id/关键词无法覆盖所有厂商

### 推荐方案：录制回放 + AOSP兜底（混合方案）
```
首次（无录制数据）：
  先尝试 AOSP 标准特征匹配（button1/spinner）→ 覆盖大多数
  匹配失败 → 提示手动操作 + 录制节点特征 → 存SP

后续（有录制数据）：
  按录制特征找节点 → performAction 回放
  特征匹配失败（系统升级）→ 清录制数据，回退首次流程
```

### 核心逻辑
- 只需适配"本机这一台设备"，不是适配全宇宙
- 录制回放天然适配本机实际弹窗结构
- AOSP标准特征(button1)作为快速通道覆盖90%设备
- 用户尚未确认方案
