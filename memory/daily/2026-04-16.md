# 2026-04-16 浏览器自动化经验

## Chrome CDP 关键配置

- Chrome 启动必须用 `--user-data-dir` 指定非默认目录，否则 CDP 报错：
  `DevTools remote debugging requires a non-default data directory`
- 示例：`--remote-debugging-port=8899 --user-data-dir="$HOME/.chrome-custom-profile"`
- 所有自动化脚本必须共用同一个 Chrome 实例
- 开机/重启后需重新启动 Chrome（CDP 模式）
- CDP 必须先 `enable Network domain`，再使用 `getAllCookies`

## Playwright 网络加载策略

- **绝对禁止** `wait_until="networkidle"`：任何含持续流式下载的页面永远等不到
- 推荐用 `wait_until="domcontentloaded"` + `time.sleep(5)`
- 下载导航用 `wait_until="commit"`（只等服务器首次响应，<1秒）

## Playwright 下载正确用法

- 用 `expect_download()` + `wait_until="commit"`（不是 `wait_for_download`）
- 必须在 `page.goto()` **之前**调用 `expect_download()`
- CDN URL 绑定 IP 时直接下载返回 403，必须通过 Chrome 拦截
- `ctx.request.fetch(url)` 可绕过 CORS（通过 Playwright 的 CDP 代理）

## 通用经验

1. 页面含视频/音频流 → 绝不用 `networkidle`
2. 所有外部 HTTP 请求超时 → 走 Chrome CDP 是最可靠方案
3. Cookie 格式不兼容 → 直接用 CDP 导出的原始 cookie
4. 持续报错状态（如 consecutiveErrors）可能需要手动重置
