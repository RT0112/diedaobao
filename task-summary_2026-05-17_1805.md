## 核心Bug：每30秒弹一次协助请求（无限循环）

### 根因
`RemoteAssistManager.kt` 的 WS 监听器用 `fromId + currentTimeMillis()/1000` 做请求去重ID。
`System.currentTimeMillis()/1000` 每秒都变 → `lastNotifiedRequestId` 永远匹配不上 → 
每次WS推送的请求都当新请求 → 弹 Activity → 15秒自动拒绝 → 关闭 → 
健康检查30秒后重启轮询 → 又收到同一请求 → 又弹 → 无限循环

### 时间线证据
17:54~18:01，每30秒弹一次 RemoteAssistActivity：
- 17:54:16, 17:54:44, 17:55:29, 17:55:59, 17:56:29, 17:56:59, 17:57:29, 17:57:59, 17:58:29, 17:58:59, 17:59:29, 17:59:59, 18:00:29, 18:00:59, 18:01:29

### 修复
- WS 监听器：`guardianId + requestTime` 替代 `fromId + currentTimeMillis()/1000`
- HTTP poll：`fromId + requestTime` 替代 `fromId + remainingSeconds`

## Bug：ScreenCaptureService 18秒后被杀

### 根因
RemoteAssistActivity 的 `onDestroy()` 中 `if (isAssisting) cleanupAssist()` → 
cleanupAssist() 调 stopService(ScreenCaptureService) → 屏幕共享中断。
Activity 因自动拒绝倒计时/系统回收而关闭 ≠ 用户想结束协助。

### 修复
`onDestroy()` 中：如果 isAssisting=true，只解绑 Service 不停 Service，让 ScreenCaptureService 继续运行

### 修复
