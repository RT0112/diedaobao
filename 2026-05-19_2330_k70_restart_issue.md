# K70 服务端重启问题总结

## 时间：2026-05-19 23:30

## 问题
K70 上的 diedaobao-server (Node.js) 已停止，需要重启。

## 尝试过的方法（全部失败）

### 1. Termux HOME_COMMAND Intent
- 命令：`am start -n com.termux/.HomeActivity -e com.termux.extra.HOME_COMMAND '...'`
- 结果：Intent 被接收（result=0），但命令不实际执行
- 原因：HOME_COMMAND 在隔离环境中执行，无法访问 Termux 文件系统

### 2. adb shell 直接执行
- 命令：`adb shell "export HOME=... PATH=... && node server.js"`
- 结果：node 不在系统 PATH 中
- 原因：adb shell 的环境和 Termux 完全不同

### 3. Termux:API ExecuteCommand 广播
- 命令：`am broadcast -a com.termux.api.action.EXECUTE_COMMAND ...`
- 结果：广播返回成功，但命令不执行
- 原因：ExecuteCommand 在隔离沙箱中运行

### 4. 完整路径执行
- 命令：`/data/data/com.termux/files/usr/bin/node`
- 结果：文件不存在
- 原因：node 不在预期位置

### 5. 子代理执行
- 多个子代理尝试执行 ADB 命令
- 结果：全部超时（1分钟或30秒）

## 根本原因
1. **环境隔离**：adb shell 和 Termux 是完全隔离的两个环境
2. **文件系统隔离**：adb shell 无法访问 Termux 的 /data/data/com.termux/files/ 目录
3. **PATH 不同**：Termux 的 node 不在系统 PATH 中
4. **权限限制**：MIUI 限制了后台执行和跨应用访问

## 唯一可靠方案
**用户手动在 K70 上操作**：
```bash
# 在 Termux 中执行
pkill -f node
cd ~/.qclaw/diedaobao-server
nohup node server.js > server.log 2>&1 &
```
