# 远程协助两问题修复 v29 - 2026-05-26

## 目标
修复用户报告的两个远程协助可靠性问题：
1. 关闭自动允许时，协助画面突然退出、子女端卡住
2. 开启自动允许时，倒计时30秒不动、页面闪退到首页
3. 关闭自动允许时，倒计时跳太快（60-58-56）

## 根因分析

### 问题1：关闭自动允许时协助画面突然退出
**根因**：`RemoteAssistManager.kt` 的 `AssistCancel` 处理里没有调用 `onSessionEnded?.invoke()`，导致子女端取消请求时老人端不关闭请求页面。

**修复**：在 `AssistCancel` 处理里加 `onSessionEnded?.invoke()`。

### 问题2：开启自动允许时倒计时不动
**根因1**：`verifyAndFinish()` 里有 `if (!isAssisting) return`，如果协助还没开始（`isAssisting=false`），收到 `AssistEnd` 或 `AssistCancel` 后不会关闭 Activity。

**根因2**：`onNewIntent` 和 `onCreate` 的重复请求分支里没有调用 `startAutoRejectCountdown()`，导致重复请求时倒计时没有重启。

**修复**：
1. 在 `verifyAndFinish()` 里加 `isAssisting=false` 分支，直接关闭请求页面
2. 在 `onNewIntent` 和 `onCreate` 的重复请求分支里加 `startAutoRejectCountdown()`

### 问题3：倒计时跳太快
**根因**：`onNewIntent` 多次触发会创建多个 `Runnable` 共享 `remainingSeconds`，导致倒计时加速。

**修复**：已在之前版本（v28）里修复，通过 `isCountdownRunning` 防重入。

## 编译错误修复

### 错误1：`LoginActivity.kt` 重复 import
```
e: Conflicting import, imported name 'WSClient' is ambiguous
```
**修复**：删除重复的 `import com.falldetector.diedaobao.cloud.WSClient`。

### 错误2：`FallDetectionService.kt` 未解析的引用
```
e: Unresolved reference: DEDUP_WINDOW_MS
e: Unresolved reference: lastHandledRequestTime
```
**修复**：在 `RemoteAssistActivity.kt` 的 `companion object` 里加：
```kotlin
const val DEDUP_WINDOW_MS = 3000L
var lastHandledRequestTime = 0L
```
并在 `handleNewRequest()` 里更新 `lastHandledRequestTime = System.currentTimeMillis()`。

## 经验总结
1. **WS事件处理要完整**：`AssistEnd` 和 `AssistCancel` 都要调用 `onSessionEnded?.invoke()`，否则对应场景不会关闭页面
2. **`verifyAndFinish()` 要处理所有场景**：不仅要处理协助中断连，还要处理请求被取消（`isAssisting=false`）
3. **重复请求要重置倒计时**：无论是在 `onCreate` 还是 `onNewIntent`，重复请求都要取消旧的 `Runnable` 并重启倒计时
4. **编译错误要立即修复**：`duplicate import` 和 `unresolved reference` 都是低级错误，应该在提交前检查
5. **`companion object` 字段默认是 `public`**：如果加了 `private`，其他类无法访问
