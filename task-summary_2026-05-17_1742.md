## 修复的三个问题

### Bug: 子女端只看到一帧就卡住
- **根因**: `network_security_config.xml` 中 `192.168.0.0` 是 CIDR 网段，Android domain-config 做域名匹配不支持 CIDR → `http://192.168.4.19` 被 Cleartext HTTP 策略拒绝 → `ScreenCaptureService.notifyScreenReady()` 抛 IOException 崩溃进程 → 帧上传停止
- **修复**: network_security_config.xml 添加 `<domain>192.168.4.19</domain>` 具体IP白名单

### Bug: 再次请求老人端闪退
- **根因**: 同上，notifyScreenReady() 崩溃杀进程 → 重启后状态丢失 → 再次请求时状态混乱
- **修复**: notifyScreenReady() thread{} 内加 try-catch 防止未捕获异常崩溃进程

### Bug: 老人端一直弹"没有待处理的请求"
- **根因**: v24 修复导致重复请求时走 onAllowClicked() → respondToRequest(true) → 后端已 active → 返回错误 → toast 显示原文
- **修复**: 
  - v25: 重复请求只显示当前状态，不走 onAllowClicked()
  - respondToRequest 失败时：已在协助中则忽略，否则显示友好提示

## 修改文件
1. `app/src/main/res/xml/network_security_config.xml` — 添加 192.168.4.19 IP白名单
2. `app/src/main/java/com/falldetector/diedaobao/assist/ScreenCaptureService.kt` — notifyScreenReady 内部 try-catch
3. `app/src/main/java/com/falldetector/diedaobao/ui/RemoteAssistActivity.kt` — v25 重复请求处理 + respondToRequest 失败友好提示
