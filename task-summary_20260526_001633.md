# WS断线重连修复 + 地图位置刷新验证

## 时间
2026-05-26 00:01 ~ 00:16

## 目标
修复子女端地图第二次进入不刷新位置的Bug，以及WS断线不重连的根本问题

## 根因分析

### 地图位置不刷新的完整根因链
1. **老人端WS断线不重连** → 服务端 `elders: 0`
2. 子女端发 `location_request` → 服务端发现老人WS离线 → 写DB标记
3. 但老人端App还在跑（GPS定位正常），只是WS断了
4. WS请求30秒超时 → HTTP降级成功但体验差

### WS断线不重连的根因
- **老人端**: `MAX_RECONNECT_ATTEMPTS = 5`，5次失败后永久放弃
- **子女端**: `MAX_RECONNECT_ATTEMPTS = 10`，10次失败后永久放弃
- 无看门狗机制，断线后无法自动恢复
- 老人端额外问题：`@Volatile`缺失、`replay=1`未改、`onClosing`未清webSocket

## 修复内容

### 老人端 WSClient.kt (fall-detection-app v0.45.8, versionCode=150)
1. **指数退避无限重连**: 3s→6s→12s→24s→48s→60s封顶，永不放弃
2. **2分钟看门狗**: 定期检查WS状态，断线自动触发重连（重置退避计数）
3. **@Volatile注解**: 修复 `isConnected`/`webSocket` 线程不安全
4. **replay=0**: 防止旧LocationUpdate重放
5. **onClosing清webSocket=null**: 防止残留引用

### 子女端 WSClient.kt (family-guardian-app v0.9.4, versionCode=48)
1. **指数退避无限重连**: 5s→10s→20s→40s→60s封顶，永不放弃
2. **2分钟看门狗**: 定期检查WS状态，断线自动触发重连
