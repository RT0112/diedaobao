# Terminal界面可见操作方式验证

## 时间
2026-05-22 13:40

## 需求
用户要求在Mac Terminal.app界面上直接看到命令输入和输出，而不是在后台偷偷执行。

## 方案
用AppleScript控制Terminal.app + System Events keystroke：
1. `tell application "Terminal" to activate` — 激活终端
2. `tell application "System Events" to keystroke "命令"` — 模拟键盘输入
3. `key code 36` — 回车

## 关键点
- osascript会阻塞，必须background运行
- keystroke方式直接在用户可见的Terminal窗口里打字
- 用户能看到完整的输入和输出过程
- CC命令也可以用这种方式输入

## 代码模板
```bash
osascript -e '
tell application "Terminal"
    activate
end tell
delay 0.5
tell application "System Events"
    keystroke "cd ~/.qclaw/workspace-x5kuz49xple53hhg/projects && bash anthropic-qclaw-proxy/cc-agent.sh \"任务描述\""
    key code 36
end tell
'
```
需要 background=true 执行。

## 验证结果
✅ 测试echo命令成功，keystroke能往Terminal窗口输入文字并回车
