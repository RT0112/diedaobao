# 任务总结：修复远程协助倒计时过短问题

## 问题
用户反馈：老人端弹出的"允许协助"页面倒计时没几秒就关闭了，要求设置为60秒。

## 根因分析
1. **后端server.js**：`remainingSeconds` 根据 `elapsed` 时间动态计算（`Math.ceil((60000 - elapsed) / 1000)`），网络延迟会导致老人端收到的倒计时已经减少了
2. **RemoteAssistActivity.kt**：默认值 `remainingSeconds = 30`（不是60）
3. **FallDetectionService.kt**：通知超时 `setTimeoutAfter(request.remainingSeconds * 1000L)` 跟随后端计算值，导致通知提前消失

## 修复内容

### 1. 后端 `server.js`
```javascript
// 修改前：remainingSeconds: Math.ceil((60000 - elapsed) / 1000)
// 修改后：固定返回60
remainingSeconds: 60
```

### 2. 老人端 `RemoteAssistActivity.kt`
```kotlin
// 修改前：private var remainingSeconds = 30
// 修改后：
private var remainingSeconds = 60

// 修改前：remainingSeconds = extras?.getInt("remaining_seconds", 60) ?: 60
// 修改后：
remainingSeconds = 60  // 固定60秒，不依赖后端传入值
```

### 3. 老人端 `FallDetectionService.kt`
```kotlin
// 修改前：.setTimeoutAfter(request.remainingSeconds * 1000L)
// 修改后：
.setTimeoutAfter(60_000L)  // 固定60秒，不随remainingSeconds变化
```
