# 任务摘要：远程协助页面"隔一次闪退"根因分析

## 目标
分析老人端远程协助请求页面第1、3、5次闪退（闪一下就消失）、第2、4次正常的交替闪退问题。

## 根因
**上一次会话的 `assist_end` WS信号延迟到达，杀掉了新会话的 Activity。**

因果链：
1. 子女端结束协助 → WS发 `assist_end` → 老人端正常关闭Activity ✅
2. 子女端立即再次请求 → WS发 `assist_request` → 老人端打开新Activity ✅
3. 但后端/子女端的旧会话 `end` HTTP请求还在飞 → 后端再次推送 `assist_end` WS
4. 老人端 `onSessionEnded` 回调仍绑定 → 触发 `cleanupAssist()` + `finish()` → 新Activity被杀 💥

关键代码缺陷：
- `RemoteAssistManager.onSessionEnded` 是全局var，无会话隔离
- 后端 `action=end` 双端推送 `assist_end`（L629-635），不检查会话状态
- 子女端 `requestAssist()` 先发 `sendAssistEnd` 清旧会话（L153），产生额外end信号

## 推荐方案
**方案1+2组合**：
- 方案2（快速止血）：新Activity启动后3秒内忽略assist_end信号
- 方案1（正确架构）：sessionId匹配，只响应当前会话的end信号
