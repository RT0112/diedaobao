## 关键经验（新增）

### K70数据库操作限制
- **sqlite3 CLI不可用**：Termux环境下sqlite3命令不存在
- **better-sqlite3模块未安装**：Node.js直接操作数据库需用 `node:sqlite` 内置模块
- **必须通过Node.js操作**：创建 `create_feedback_table_v2.js` 脚本，通过 `node` 命令执行

### SQL语法陷阱
- **SQLite字符串字面量用单引号**：双引号 `"unixepoch"` 会被识别为列名，导致错误 `no such column: "unixepoch"`
- **正确写法**：`datetime(timestamp/1000, 'unixepoch', 'localtime')`（单引号）

### 后端路由调试
- **先grep确认路由存在**：`ssh -p 8022 -i ~/.ssh/k70_key u0_a280@localhost "grep -n 'feedback' ~/diedaobao-server/server.js"`
- **404错误先检查路由注册**：确认 `app.post('/feedback', ...)` 在 `app.use((req, res) => { res.status(404)... })` 之前

### URL配置策略
- **测试环境**：本地IP `http://192.168.4.19:3000`（同一WiFi下可访问）
- **生产环境**：隧道URL（cloudflare tunnel或serveo.net SSH反向隧道）
- **避免硬编码**：使用 `CloudBaseClient.BASE_URL` 统一管理
