# 任务摘要 - GitHub SSH Key配置与代码推送

## 时间
2026-05-20 14:07 ~ 14:15

## 目标
解决GitHub推送认证问题（之前PAT创建失败），改用SSH key方式

## 关键步骤
1. **xbrowser点击"Add SSH key"按钮** — 之前多次click操作失败，最终用JavaScript `(function(){...})()` 包裹eval成功触发点击
2. **页面跳转确认成功** — URL从 `/settings/ssh/new` 跳转到 `/settings/keys`，页面显示 "You have successfully added the key 'diedaobao-mac'"
3. **SSH配置** — 私钥文件为 `~/.ssh/github_diedaobao`（非默认id_ed25519），需配置 `~/.ssh/config` 指定IdentityFile
4. **known_hosts** — 需要先 `ssh-keyscan github.com >> ~/.ssh/known_hosts`
5. **remote URL切换** — 从HTTPS改为SSH格式：`git@github.com:RT0112/diedaobao.git`
6. **推送成功** — `git push -u origin main` 一次成功

## 踩过的坑
- xbrowser的click操作多次超时/无响应 → 用eval + IIFE JavaScript click()解决
- SSH默认找 `id_ed25519` 但私钥叫 `github_diedaobao` → 必须配config
- 首次连接需要先加known_hosts否则Host key verification failed

## 经验提炼
- GitHub网页表单提交用xbrowser不可靠时，改用eval+JS click()是有效fallback
- SSH key方式比PAT更适合长期使用（不过期、不用管理token）
- 私钥文件名非标准时必须配 ~/.ssh/config
