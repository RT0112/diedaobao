## 模块1：电子围栏（Geofence）

> 用户提出的核心功能之一

### 1.1 功能概述

家属可为老人设定安全活动范围，一旦老人超出范围，APP立即推送提醒给家属，防止走失。

### 1.2 核心功能

| 功能 | 描述 |
|------|------|
| 围栏创建 | 支持圆形围栏（中心点+半径）和多边形围栏 |
| 围栏管理 | 可创建多个围栏，命名（如"家"、"公园"），设置生效时间段 |
| 越界检测 | 实时监测老人位置，离开围栏立即触发通知 |
| 通知推送 | 通过企业微信/短信通知家属，包含老人当前位置链接 |
| 历史记录 | 记录越界事件，包含时间、位置、围栏名称 |

### 1.3 技术实现

**Android端**：
- 使用 `GeofencingClient` (Google Play Services) 或自建位置监测
- 后台持续定位：`ForegroundService` + `FusedLocationProviderClient`
- 定位策略：围栏内低频（5分钟/次），围栏边界高频（30秒/次）

**数据结构**：
```kotlin
@Entity
data class GeofenceZone(
    @PrimaryKey val id: String,
    val name: String,           // 围栏名称
    val centerLat: Double,      // 圆心纬度
    val centerLng: Double,      // 圆心经度
    val radiusMeters: Float,    // 半径（米）
    val isActive: Boolean,      // 是否启用
    val notifyOnExit: Boolean,  // 离开时通知
    val notifyOnEnter: Boolean, // 进入时通知
    val createdAt: Long
)
```

**首页卡片**：

## 模块2：亲情守护端（Family Guardian）

> 用户提出的核心功能之一，整合了跌倒后状态追踪、轨迹查询

### 2.1 功能概述

子女通过独立APP或微信小程序，远程查看老人位置、跌倒记录、健康数据、历史轨迹，实现远程关爱。

### 2.2 核心功能

| 功能 | 描述 |
|------|------|
| 实时位置 | 查看老人当前位置（地图展示） |
| 跌倒记录 | 查看历史跌倒事件，包含时间、位置、处理状态 |
| 跌倒后追踪 | 报警后持续监测老人手机状态（位置、运动、电量） |
| 健康概览 | 查看老人健康数据（血压、血糖、心率） |
| 设备状态 | 查看老人手机电量、网络状态、APP运行状态 |
| 轨迹查询 | 查看历史位置轨迹，回溯老人行踪 |
| 远程设置 | 远程配置围栏、提醒、紧急联系人 |
| 紧急呼叫 | 一键发起语音通话，确认老人状况 |

### 2.3 跌倒后状态追踪（核心子功能）

**状态追踪流程**：
```
跌倒检测触发
  ↓
发送报警通知
  ↓
进入"追踪模式"（30分钟）
  ├─ 每30秒获取位置
  ├─ 持续监测加速度（判断是否移动）
  ├─ 监测手机电量
  └─ 推送状态到家属端
  ↓
老人确认"我没事" → 结束追踪
或
30分钟无响应 → 紧急升级
```

### 2.4 技术架构

**方案A：独立APP（推荐）**
- 子女端独立APP，通过账号体系关联老人设备
- 老人端定期上报位置/状态到服务端
- 子女端实时拉取或WebSocket推送

**方案B：微信小程序**
- 无需安装，扫码即用
- 通过微信账号关联老人设备

**数据同步**：
```kotlin

    val deviceId: String,
    val location: Location?,
    val battery: Int,
    val isCharging: Boolean,
    val networkType: String,
    val appRunning: Boolean,
    val lastFallTime: Long?,
    val timestamp: Long
)
```

## 模块3：日常提醒（Daily Reminders）

> 用户提出的核心功能之一，整合了用药提醒

### 3.1 功能概述

支持吃药、喝水、睡觉、活动、测血压等多种日常提醒，帮助老人养成健康生活习惯。

### 3.2 核心功能

| 提醒类型 | 默认设置 | 自定义选项 |
|----------|----------|------------|
| 用药提醒 | 按医嘱设置 | 药品名称、剂量、时间、语音播报 |
| 喝水提醒 | 每2小时一次 | 间隔时间、时间段、每日目标 |
| 睡觉提醒 | 22:00 提醒就寝 | 时间、是否开启 |
| 起床提醒 | 07:00 提醒起床 | 时间、是否开启 |
| 活动提醒 | 每1小时提醒起身活动 | 间隔时间、时间段 |
| 测血压 | 08:00、20:00 | 时间、频率 |
| 测血糖 | 饭后2小时 | 时间、频率 |

### 3.3 用药提醒（核心子功能）

| 功能 | 描述 |
|------|------|
| 药物管理 | 录入药品信息：名称、剂量、频率、注意事项、图片 |
| 提醒设置 | 设置提醒时间（支持每天/隔天/每周） |
| 提醒方式 | 语音播报（"该吃降压药了"）+ 弹窗 + 震动 |
| 服药确认 | 老人点击"已服用"确认，记录服药时间 |
| 漏服提醒 | 15分钟后未确认，再次提醒并通知家属 |

### 3.4 技术实现

**数据结构**：
```kotlin
@Entity

    WATER,      // 喝水

    ACTIVITY,   // 活动
    BLOOD_PRESSURE,  // 血压
    BLOOD_SUGAR,     // 血糖

```

## 模块5：语音助手（Voice Assistant）

> 新增功能 - 简化老人操作

### 5.1 功能概述

通过语音控制APP，老人无需复杂点击操作，直接说话即可完成功能。

### 5.2 核心功能

| 功能 | 描述 |
|------|------|
| 语音唤醒 | "小宝小宝"唤醒语音助手 |
| 语音拨号 | "给儿子打电话" → 自动拨号 |
| 语音提醒 | "提醒我吃药" → 设置提醒 |
| 语音查询 | "今天要吃什么药" → 语音播报 |
| 语音导航 | "打开地图" → 跳转地图页面 |
| 语音求助 | "救命" → 触发紧急求助 |

### 5.3 技术实现

**Android语音识别**：
- 使用 `SpeechRecognizer` API（离线可用）
- 关键词匹配：预定义指令集
- 语音合成：`TextToSpeech` 播报结果

**指令集**：
```kotlin
val voiceCommands = mapOf(
    "给儿子打电话" to { callFamily("儿子") },
    "给女儿打电话" to { callFamily("女儿") },

    "今天要吃什么药" to { speakMedicationList() },
    "救命" to { triggerEmergency() },
    "我没事" to { cancelFallAlert() }
)
```

## 模块7：紧急求助增强（Enhanced SOS）

> 新增功能 - 强化紧急求助能力

### 7.1 功能概述

除跌倒自动报警外，增加手动一键SOS、自动报警升级、多渠道通知等功能。

### 7.2 核心功能

| 功能 | 描述 |
|------|------|
| 一键SOS | 首页大红色按钮，长按3秒触发紧急求助 |
| 多渠道通知 | 电话 + 短信 + 企业微信 + 邮件（可选） |
| 自动升级 | 30分钟无响应，自动升级：通知更多人或拨打110 |
| 循环呼叫 | 紧急联系人未接听，自动呼叫下一个 |
| 位置共享 | 求助时自动分享位置链接给所有联系人 |
| 求助记录 | 记录所有求助事件，包含处理状态 |

### 7.3 技术实现

**一键SOS流程**：
```
长按SOS按钮3秒
  ↓
震动反馈 + 倒计时提示
  ↓
触发紧急求助
  ├─ 获取当前位置
  ├─ 发送通知给所有紧急联系人
  ├─ 自动拨打第一个联系人
  └─ 进入"等待响应"状态
  ↓
30分钟无响应
  ├─ 自动升级：通知更多联系人
  └─ 可选：自动拨打110
```

**数据结构**：
```kotlin
@Entity
data class SosEvent(
    @PrimaryKey val id: String,
    val triggerType: String,    // manual/fall/upgrade
    val location: Location?,
    val notifiedContacts: List<String>,
    val callAttempts: List<CallAttempt>,
    val respondedAt: Long?,
    val resolvedAt: Long?,
    val resolution: String?     // "safe"/"help_arrived"/"escalated"
)
```

## 模块8：远程协助（Remote Assistance）

> 新增功能 - 子女帮老人操作手机

### 8.1 功能概述

子女远程控制老人手机，帮助老人完成复杂操作（如设置、安装应用等）。

### 8.2 核心功能

| 功能 | 描述 |
|------|------|
| 屏幕共享 | 老人手机屏幕实时共享给子女 |
| 远程控制 | 子女远程操作老人手机 |
| 语音引导 | 子女语音指导老人操作 |
| 文件传输 | 子女远程给老人手机发送文件 |
| 应用安装 | 子女远程帮老人安装应用 |

### 8.3 技术实现

**方案A：使用第三方SDK（推荐）**
- 集成 `AnyDesk` 或 `TeamViewer` SDK
- 成熟稳定，开发成本低
- 需要商业授权

**方案B：自建远程协助**
- 使用 WebRTC + VNC
- 开发成本高
- 完全自主可控

**安全机制**：
- 老人需授权同意才能远程控制
- 每次远程协助需确认
- 敏感操作需二次确认

## 亲情守护端详细规划（模块2）

> 用户提到"内容大要好好规划"

### 架构选择

**方案对比**：

| 方案 | 优点 | 缺点 | 推荐度 |
|------|------|------|--------|
| 独立APP | 功能完整、体验好 | 需要安装 | ⭐⭐⭐⭐⭐ |
| 微信小程序 | 无需安装、易推广 | 功能受限 | ⭐⭐⭐⭐ |
| H5网页 | 跨平台、易分享 | 体验差 | ⭐⭐ |

**推荐方案**：独立APP（主） + 微信小程序（辅）

### 服务端架构

```
老人端APP
    ↓ 上报状态
服务端（云服务器）
    ├─ 用户账号体系
    ├─ 设备绑定关系
    ├─ 实时状态存储（Redis）
    ├─ 历史数据存储（MySQL）
    └─ WebSocket推送
    ↓
子女端APP/小程序
```

### 数据同步策略

| 数据类型 | 同步频率 | 存储 |
|----------|----------|------|
| 实时位置 | 每5分钟或围栏边界触发 | Redis（7天过期） |
| 跌倒事件 | 即时推送 + 本地存储 | MySQL（永久） |
| 健康数据 | WiFi下批量同步 | MySQL（永久） |
| 轨迹数据 | 每30分钟批量上传 | MySQL（7天后清理） |

### 隐私保护

- 端到端加密：位置数据加密传输
- 授权机制：老人端需明确授权子女查看权限
- 可撤销：老人可随时取消子女的查看权限
- 数据留存：轨迹数据仅保留7天，自动清理

### 开发工时估算

| 模块 | 工时 | 说明 |
|------|------|------|
| 老人端上报逻辑 | 2天 | 位置、状态、跌倒事件上报 |
| 服务端API | 3天 | 用户、设备、数据接口 |
| 子女端APP | 5天 | 首页、地图、跌倒记录、健康数据 |
| 微信小程序 | 3天 | 核心功能移植 |
| 测试联调 | 2天 | 端到端测试 |
| **总计** | **15天** | |
